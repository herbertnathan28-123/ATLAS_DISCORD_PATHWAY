## Cursor Cloud specific instructions

### Services overview

This repo is a single Node.js process (`node index.js`) that runs:
- **Discord bot** (FX Bot#3867) — command handler + message delivery
- **TrendSpider HTTP server** (port 3001) — webhook receiver + `/health` + `POST /atlas/run`
- **Dark Horse Engine** — 15-minute scanner/scheduler
- **Corey Market Intel** — calendar + daily bulletin scheduler

All services start from `index.js`. There is no separate backend/frontend split.

### Running locally (QA / dev mode)

The bot requires three fatal env vars: `DISCORD_BOT_TOKEN`, `TWELVE_DATA_API_KEY`, `SYSTEM_STATE`.
For offline QA without external APIs or Discord login:

```bash
ATLAS_NO_LOGIN=1 DISCORD_BOT_TOKEN=qa-stub TWELVE_DATA_API_KEY=qa-stub \
  SYSTEM_STATE=BUILD_MODE node index.js
```

This boots the full module graph (Corey, Dark Horse, Macro v3, calendar)
but suppresses Discord login and the TrendSpider HTTP listener (which
only starts after the Discord `clientReady` event).

### Tests

- **Recovery acceptance suite** (CI gate, no env vars needed): `node scripts/test_recovery.js`
- **Doctrine audit** (static wiring check): `npm run audit`
- **QA scripts** (stub-token safe): `npm run qa:live-route`, `npm run qa:discord-batch`, `npm run qa:darkhorse`, `npm run qa:wording-directives`, `npm run qa:macro-operational`
- **Tests directory** (`tests/*.test.js`): standalone Node scripts (no test framework), run individually with `node tests/<file>.test.js`

There is no ESLint, Prettier, or formal linter configured. Use `node -c <file>` for syntax checks.

### Node.js version

`package.json` declares `"engines": { "node": "20.x" }`. Use nvm: `nvm use 20`.

### Gotchas

- Puppeteer Chrome must be installed after `npm install` via `npx puppeteer browsers install chrome`.
- The `ATLAS_NO_LOGIN=1` mode does **not** start the HTTP server on port 3001 — the server only starts inside the `clientReady` handler, which requires a real Discord login.
- API key errors in Corey live data logs are expected when using stub tokens — the bot degrades gracefully.
- `audit.json` and `audit.summary.md` are generated by `npm run audit` in the repo root; they are not committed.
